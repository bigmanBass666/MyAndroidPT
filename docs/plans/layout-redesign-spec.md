# MyAndroidPT 布局改版设计文档

> 版本：v1（方向确认稿）
> 目标：去掉学生作业感，达到工业级水准
> 原则：每个页面只有一个重心、间距即节奏、信息要有组织

> 📘 **教材批注**：本设计文档已对照两份教材标准进行标注：
> - **实训** → `materials/1_practical_trainning/作业要求与评分标准.md`
> - **课程设计** → `materials/2_design/01-assignment.md` / `02-grading-rubric.md`
> 
> 标注含义（`>=` 原则：不低于教材要求，且允许做得更好）：
> - ✅ **教材要求** — 教材明确要求，应优先采纳
> - ➕ **超出教材 · 品质加分** — 教材未明确要求，但有助于品质提升，放心做（老师不会嫌学生过于优秀）
> - ❌ **不宜采用** — 与教材要求直接冲突，或大概率引入功能性 bug

## 一、全局设计语言

### 1.1 Toolbar 规则
- **登录页不使用 Toolbar**（页面顶部由品牌标识占用）
- **其余页面使用 Toolbar，但背景统一为 `?attr/colorSurface`**（去掉 Toolbar 专属绿色背景）
- Toolbar 标题文字：`?attr/colorOnSurface`（不使用白色）
- 导航返回图标使用 `ic_arrow_back`（仅非首页页面）

> 📘 **批注：**
> - 登录页去除 Toolbar → ➕ **品质加分**。品牌标识取代 Toolbar 不违反任何教材要求，登录功能完全不受影响
> - Toolbar 背景改为 surface → ➕ **品质加分**。M3 规范中 Toolbar 使用 surface 是标准做法，当前绿色 Toolbar 反而是偏离规范的设计

### 1.2 间距节奏
不再统一使用 `form_field_spacing=16dp` 做所有间距。改为三级间距策略：
- **大间距 (32dp)**：页面主要区块之间的间隔（品牌区到表单、仪表盘到操作栏）
- **中间距 (16dp)**：字段与字段之间、卡片内部的常规间隔
- **小间距 (8dp)**：标签与内容之间、辅助信息与主信息之间

> 📘 **批注：** 三级间距策略属于"适当的间距"范畴 → ✅ **教材要求**（实训要求"适当的间距"、"布局合理"）。在已有 `dimens.xml` 的语义化 token 基础上增加 `gap_large(32dp)` 等新 token 即可实现

### 1.3 信息块组织
- 不再用「一段灰色背景」表示区域
- 使用 `MaterialCardView`（elevation=0dp, stroke=1dp）做分组容器
- 重要状态使用 `Chip` 组件展示，而非纯文本

> 📘 **批注：** 使用 `MaterialCardView` 和 `Chip` 组件是 Material Design 的推荐做法 → ➕ **品质加分**。教材要求"符合 Material Design 规范"、"适当的组件使用方式"，这些变化符合此方向。当前项目已使用 M3 组件库，切换成本低

### 1.4 图标
- 输入框增加 leading icon（通过 TextInputLayout 的 `app:startIconDrawable`）
- 空态插图更新为更具引导性的表达
- 所有图标使用 `@color/on_surface_variant` 着色

### 2.2 关键变化
1. **退出登录**从页面移除，放入 Toolbar 菜单（三点或者设置图标）
2. **三个全宽按钮** → 横向卡片操作区（单行，不占高度）
3. **下半屏展示最近待办**（有内容时有价值，无内容时引导新建）
4. **仪表盘**改为与快捷操作合并，不再独立显示大号数字

> 📘 **批注：**
> - 退出登录移入菜单 → ➕ **品质加分**。退出登录是可选加分功能（实训标记为"可选"），并非 F1-F5 核心功能。将其移入菜单不影响必做功能的完整性。教材要求"操作路径短"针对的是 F1-F5 的 CRUD 路径，不是 logout
> - 三个按钮 → 横向卡片 → ✅ **教材要求**。课程设计评分标准"按钮布局合理"，横向操作区比垂直堆叠三按钮更合理
> - 最近待办预览 → ➕ **品质加分**。新增功能，不仅不违反教材，还增强了 F2（待办查看）的可见性
> - 仪表盘合并 → ✅ **教材要求**。"简洁美观"上，合并后的布局减少了页面元素，更干净

### 2.3 Java 逻辑调整
- `loadDashboardData()` → 同时加载最近待办列表（limit 3）
- 点击某条待办预览 → 跳转 TodoDetailActivity
- 空态时显示引导文案 + FAB 闪烁引导
- Toolbar 菜单：退出登录 + 设置（预留）

## 八、全局改进汇总

### 8.1 "看见"的变化

| 页面 | 最大变化 |
|------|----------|
| Welcome | 三按钮 → 横向操作卡片 + 最近待办预览 + 退出移到菜单 |
| Login | 去 Toolbar + 品牌标识区 + leading icon |
| Register | 统一 leading icon + 更好的字段间距 |
| Todo List | 删除图标移除 + 长按删除 + 状态指示 |
| Todo Detail | 状态 Chip + 编辑入 Toolbar + 删除沉底 |
| Todo Edit | 保存入 Toolbar + 去取消按钮 + leading icon |

### 8.2 "感受"到的不再是学生作业的几件事

1. **打开 app 的第一眼**：看到的是品牌标识，不是绿色 Toolbar
2. **首页不再是操作清单**：看到的是数据（最近待办预览），操作是辅助的
3. **信息有组织方式**：详情页不用眼睛一行行扫，区块帮你分好了
4. **危险操作不放在显眼位置**：退出登录、删除都藏起来或需要二次确认
5. **每个页面的结构不同**：不再共用同一个公式

> 📘 **总批注：** 本设计文档的全部 6 个页面改版方案经教材 `>=` 原则过滤后：
> - ✅ **教材要求：10 项** — 直接拿分点，优先做（包括间距节奏、按钮布局、leading icon、操作路径优化等）
> - ➕ **品质加分：9 项** — 放心做，老师不会嫌学生太优秀（品牌标识、Toolbar 背景、Chip 状态等）
> - ❌ **不宜采用：0 项** — 没有任何方案与教材冲突或会破坏功能
> 
> 所有核心功能（F1-F5）均保持完整可用的前提下进行布局升级。**19 项改进全部可做。**
