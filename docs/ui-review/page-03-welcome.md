# UI 精进：欢迎页 (`activity_welcome.xml`)

> 截图：`screenshots/03_welcome.png`

---

## 当前布局摘要

| 组件 | ID | 说明 |
|------|-----|------|
| 欢迎文案 | `tv_welcome` | TextView，显示"欢迎回来，{userName}" |
| 进入待办 | `btn_todo_list` | MaterialButton，全宽，`@style/MyBtnStyle` |
| 退出登录 | `btn_logout` | MaterialButton，TextButton 样式（无填充色） |

---

## UI 问题与改进建议

### P0 — 页面内容过于单薄

**问题**：欢迎页只有 1 个 TextView + 2 个按钮，占不满一屏。顶部大片空白，下方内容空洞。作为登录后的"首页"，用户对此页面预期是提供信息总览。

**建议**：
- 重新定位欢迎页：将其直接作为待办列表页的"过渡页"（或称 Dashboard），增加功能摘要信息
- 或直接移除欢迎页，登录后跳转待办列表（更精简的流程）

### P1 — 缺乏品牌和装饰元素

**问题**：纯文字 + 按钮，没有任何图标、卡片或品牌色块来丰富界面。

**建议**：
- 在顶部添加 App Logo/图标（如 CheckList 风格图标）
- 添加一个统计摘要卡片（如"待完成：N 项 | 已完成：M 项"）
- 使用 MaterialCardView 包裹各功能区，增加阴影层次

### P1 — 退出登录按钮样式不突出

**问题**：`btn_logout` 使用 TextButton 样式，视觉权重低。退出登录是危险操作，应有更明确的视觉提示（红色或警告色）。

**建议**：将退出按钮改为带警告色的 OutlinedButton，或使用 MaterialButton 的 `app:backgroundTint="@color/red"`。

### P2 — 按钮垂直分布

**问题**：两个按钮垂直堆叠，"进入待办列表"和"退出登录"排在同一个区域。用户进入欢迎页习惯性会点"进入待办"，但两个按钮在同一层级。

**建议**：将"退出登录"放在页面底部（靠近边缘）并用更小的字号，降低误触概率。主要操作"进入待办"居中醒目展示。

### P2 — 缺乏快速操作入口

**问题**：用户想新增待办必须"进入待办列表 → 点击 FAB"，中间多了一步。

**建议**：在欢迎页添加"快速新增待办"按钮或浮动入口，提升效率。

---

## 精进方向总结

| 优先级 | 项 | 难度 | 影响面 |
|--------|----|------|--------|
| P0 | 内容充实/重新定位 | 高 | 核心体验 |
| P1 | 品牌图标 + 统计卡片 | 中 | 视觉质感 |
| P1 | 退出按钮红色警示 | 低 | 安全提示 |
| P2 | 按钮布局分层 | 低 | 防误触 |
| P2 | 快速新增入口 | 低 | 效率 |
