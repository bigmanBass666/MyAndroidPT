#### 1.2 问题详情

**问题 1（P2）：`MyBtnStyle` 已被架空，无任何布局引用**

- `styles.xml` 第 24-27 行定义了 `MyBtnStyle`，但 `activity_main.xml` 中的登录按钮使用 `Widget.Material3.Button.OutlinedButton`，`activity_register.xml` 中的注册按钮使用 `Widget.Material3.Button`，均未引用 `MyBtnStyle`。
- 教材在实训 1 步骤 6 明确标注"可结合后续实训中 MyBtnStyle / MyEditStyle 的样式来绘制"。
- 建议：将 `MyBtnStyle` 属性合并到 MaterialButton 的样式中，或在布局中显式引用 `style="@style/MyBtnStyle"`（同时适配 MaterialButton 的 background 机制）。

**问题 2（P2）：`MyEditStyle` 已被架空，无任何布局引用**

- 同上，所有输入框使用 `TextInputLayout` + `TextInputEditText`，未引用 `MyEditStyle`。

**问题 3（P2）：`btn_bg_selector.xml` 按压态功能不全**

- 第 3-5 行：正常态和按下态使用同一颜色 `@color/colorPrimary`（#8BC34A）。教材要求按下时变色（`colorPrimaryDark` → `colorPrimary`）。
- 当前实现下用户点击按钮无视觉反馈，不符合教材"按压变色"的设计目标。
- 此外第 4 行引用了 `@color/colorPrimary`，这是 `colors.xml` 中定义的别名（`<color name="colorPrimary">@color/green_500</color>`），实际无问题，但 `@color/colorPrimaryDark` 已不存在（M3 迁移后删除）。

**问题 4（P2）：`colors.xml` 缺少 `colorAccent`**

- 教材 colors.xml 定义了 `colorAccent` = `#F4511E`。当前 colors.xml 无此颜色。
- 虽然 M3 使用 Secondary/Tertiary 色彩系统替代了 accent，但教材明确列出了这项产出。建议保留定义或在报告中说明替代关系。

## 第七部分：教材偏离汇总（撰写课程设计报告时必须说明的事项）

以下偏离建议在课程设计报告中明确解释，以展示对教材的深入理解：

|--------|---------|---------|-------------|
| MySQL → SQLite | 实训3/4 要求 MySQL + JDBC | SQLite + SQLiteOpenHelper | "本地开发环境限制，迁移至 SQLite，CRUD 逻辑等价" |
| Theme.MaterialComponents → M3 | 实训1 要求 MaterialComponents | Material3 DayNight | "升级至 Material 3 设计规范，更现代的 UI 表现" |
| `colorPrimaryDark`/`colorAccent` 移除 | 实训1 colors.xml 模板 | M3 色板（Secondary/Tertiary） | "M3 使用新的色彩体系替代旧版 Dark/Accent" |
| `MyBtnStyle`/`MyEditStyle` 未直接引用 | 实训1 要求布局引用 | 改用 M3 原生组件样式 | "组件升级后使用 Material 原生样式获得了更好的兼容性" |

| 无需注册回填 email | 实训6 仅回传 userName/password | 额外回传 email | "扩展功能：邮箱注册并回填" |
