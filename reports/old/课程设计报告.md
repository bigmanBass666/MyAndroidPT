## 一、项目背景

日常生活中，人们频繁使用待办事项工具来管理日程与任务。在 Android 移动平台上，待办应用是最经典的学习型项目之一，能够覆盖 Activity 生命周期、SQLite 持久化、UI 交互等核心知识点。

本课程设计以"基于安卓的简易待办应用开发"为主题，首先在已完成登录注册功能的基础上（包含登录验证、注册、记住密码、自动登录四项核心能力），实现完整的待办管理功能（创建、查看、编辑、删除、状态切换）。通过本项目，系统掌握 Android 原生开发流程、SQLite 数据持久化机制以及 Activity 间通信方式，为后续移动应用开发打下扎实基础。

## 三、设计思路

### 3.1 模块划分

| 模块 / 类 | 职责 |
|-----------|------|
| `MainActivity` | 登录入口（已有），包含账号密码输入、记住密码、自动登录 |
| `RegisterActivity` | 注册页（已有），账号密码确认、协议勾选、回传到登录页 |
| `WelcomeActivity` | 登录成功后欢迎页，待办列表和退出登录入口 |
| `TodoListActivity` | 待办主列表，RecyclerView 展示 + FAB 新增 + 条目点击/长按 |
| `TodoEditActivity` | 新增/编辑待办（共用），通过 EXTRA_TODO_ID 区分模式 |
| `TodoDetailActivity` | 待办详情页，展示完整信息 + 编辑/删除/状态切换 |
| `TodoDBHelper` | SQLiteOpenHelper，管理 todo 表的建表、升级和 CRUD |
| `TodoDao` | 待办数据访问层，封装 insert/update/delete/query 操作 |
| `Todo` | 待办实体类，字段：id、title、content、isDone、createTime |
| `TodoAdapter` | RecyclerView 适配器，绑定数据和处理点击/状态事件 |

### 3.2 涉及知识点

| 知识点 | 具体应用 |
|--------|----------|
| Activity 生命周期 | onCreate / onResume — 列表在 onResume 时自动刷新 |
| SQLiteOpenHelper | TodoDBHelper — 创建 todo 表、onUpgrade 重建策略 |
| RecyclerView + Adapter | TodoAdapter — 列表展示、CheckBox 状态切换、长按删除 |
| Intent 传值 | 携带 todo_id 跳转详情页/编辑页，-1 表示新增模式 |
| Cursor 游标遍历 | TodoDBHelper.queryById / queryAll — 查询结果 → List 转换 |
| AlertDialog 弹窗 | 删除确认弹窗，防止误操作 |
| SharedPreferences | 记住用户登录态（isRemember / isAutoLogin / userName / password） |

## 四、功能实现步骤

### 4.1 项目创建与目录结构

基于现有 MyAndroidPT 项目扩展，在已有登录注册模块基础上新增待办管理模块。项目采用经典 MVC 分层：Activity 层处理 UI 交互，dbunit 层封装数据库操作，bean 层定义数据实体，adapter 层负责列表渲染。

### 4.2 数据库设计

todo 表包含 5 个字段，以 `_id` 自增主键为核心：

```sql
CREATE TABLE IF NOT EXISTS todo (
    _id INTEGER PRIMARY KEY AUTOINCREMENT,
    title TEXT NOT NULL,
    content TEXT,
    is_done INTEGER NOT NULL DEFAULT 0,
    create_time INTEGER NOT NULL
);
```

`create_time` 使用 `System.currentTimeMillis()` 记录毫秒级时间戳，`is_done` 以 0/1 区分未完成/已完成。

### 4.3 实体类与 CRUD 方法

`Todo` 实体类封装了 id、title、content、isDone、createTime 五个字段，提供 getter/setter。`TodoDBHelper` 继承 `SQLiteOpenHelper`，在 `onCreate` 中执行建表 SQL，提供 insert/update/updateStatus/delete/queryById/queryAll 六种操作方法，所有 CRUD 操作在子线程中执行。

### 4.4 列表页实现

`TodoListActivity` 初始化 RecyclerView 和 `TodoAdapter`，通过 `loadTodos()` 方法在子线程中查询数据库，结果通过 `runOnUiThread` 回传更新 Adapter。`onResume` 生命周期中调用 `loadTodos()` 实现返回列表时自动刷新。还实现了 `seedIfEmpty()` 方法——当数据库为空时自动插入两条示例待办，确保演示时列表不为空。

```java
private void loadTodos() {
    new Thread(() -> {
        seedIfEmpty();
        List<Todo> list = dbHelper.queryAll();
        runOnUiThread(() -> adapter.setTodos(list));
    }).start();
}
```

### 4.5 创建功能实现

`TodoEditActivity` 通过读取 Intent 中的 `EXTRA_TODO_ID` 判断当前模式：`-1` 为新增模式，其余为编辑模式。用户输入标题和内容后点击"保存"，调用 `TodoDao` 插入或更新数据库，成功后 `setResult(RESULT_OK)` 返回列表页。

```java
todoId = getIntent().getIntExtra(EXTRA_TODO_ID, -1);
if (todoId != -1) {
    loadTodo(todoId);  // 编辑模式：预填已有数据
} else {
    getSupportActionBar().setTitle("新增待办");
}
```

### 4.6 查看功能实现

列表页条目点击通过 `onItemClick` 回调跳转 `TodoDetailActivity`，携带 todo_id 参数。详情页在子线程查询数据库，绑定标题、内容、创建时间、完成状态到对应 TextView/CheckBox。

### 4.7 编辑功能实现

详情页点击"编辑"按钮跳转 `TodoEditActivity`（编辑模式），携带相同 todo_id。编辑页预填已有数据，保存时调用 `TodoDao.update()` 更新 title 和 content 字段，is_done 和 create_time 保持不变。

### 4.8 删除功能实现

详情页点击删除按钮弹出 `AlertDialog` 二次确认，确认后在子线程执行 `TodoDao.delete()`，成功后 `setResult(RESULT_OK)` 并 finish 返回列表页，列表自动刷新。

### 4.9 状态切换功能实现

列表页 CheckBox 点击触发 `onStatusChanged` 回调，在子线程执行 `TodoDao.updateStatus()` 仅更新 is_done 列；详情页 CheckBox 同样在子线程更新并同步刷新状态文本。

## 五、遇到的问题及解决办法

### 问题 1：模拟器 Activity 截图时前台焦点丢失，截图抓到桌面而非应用界面

**原因分析**：在自动化验证阶段，通过 `am start` 或 `android run --activity` 切换 Activity 后，Android 系统会立即将应用置于后台（Launcher 抢占前台）。此时立即截图会捕获到手机桌面而非目标 Activity，导致视觉分析结果无效。经过多轮尝试发现，这是 Android 系统行为而非代码 bug。

**解决方案**：在每次 Activity 切换后，使用 ADB monkey 命令强制将应用锁定到前台：

```bash
adb shell monkey -p com.ljx.pt -c android.intent.category.LAUNCHER 1
```

该命令以 LAUNCHER 类别启动应用，确保 Activity 保持在前台。经过验证，此方法可稳定保持 Activity 前台状态，截图能正确捕获应用界面。

### 问题 2：模拟器环境无预置数据，TodoListActivity 列表为空

**原因分析**：模拟器是干净环境，todo.db 数据库首次创建时没有任何待办数据。`TodoListActivity` 在空列表状态下截图，无法展示应用核心功能（列表渲染、条目样式、FAB 交互）。

**解决方案**：在 `TodoListActivity.loadTodos()` 中增加 `seedIfEmpty()` 方法，首次查询时检测列表是否为空，若为空则自动插入两条示例待办（"学习Android开发"、"提交项目代码"），确保演示环境始终有数据展示。此方法不影响正常用户使用流程（用户注册后自行创建的待办优先显示）。

```java
private void seedIfEmpty() {
    if (dbHelper.queryAll().isEmpty()) {
        Todo t1 = new Todo();
        t1.setTitle("学习Android开发");
        t1.setContent("完成实训报告和课程设计报告");
        dbHelper.insert(t1);
        // ...
    }
}
```

### 问题 3：中文字符输入受限，ADB input text 触发系统异常

**原因分析**：测试阶段尝试通过 `adb shell input text` 向模拟器输入中文，由于 Android 系统对中文输入的处理机制，调用触发 NullPointerException 异常。这是 ADB input text 工具本身的已知限制，并非应用代码问题。

**解决方案**：放弃了 ADB 自动化输入的方案，转而采用代码层面的种子数据注入（seedIfEmpty）来预置演示数据。登录和注册流程通过在 MainActivity 和 RegisterActivity 的布局上直接操作完成视觉验证，不依赖自动化输入。

## 七、总结

通过本课程设计，系统掌握了 Android 原生开发的完整流程。技术上，深入理解了 Activity 间跳转与回传机制、SQLite 数据持久化、SharedPreferences 本地缓存、RecyclerView 列表渲染等核心知识点。从编码角度，掌握了子线程数据库操作 + `runOnUiThread` 回传 UI 的标准模式，理解了 Intent 传值和 ActivityResult 的区别。

项目还存在一些不足之处：界面美化程度有限，未使用 Material Design 组件库；密码以明文存储存在安全风险；未引入网络请求框架，与后端交互简陋。后续改进方向包括引入 OkHttp/Retrofit 替代裸 JDBC、实现密码哈希存储、引入 Material Components 升级界面、增加待办分类和提醒功能。
