# UI 精进 Spec

## 为什么

App 虽然使用了 Material 3 组件，但整体视觉效果平平——输入框风格不一致、空态简陋、密码框无切换按钮、缺少视觉层次。在提交给老师的课程设计项目中，UI 质感直接影响 20/100 分的界面设计评分。

## 改动内容

**范围**：6 个页面，聚焦 P0（必修）+ 部分 P1（建议），跳过 P2/P3

- 密码可见性切换（登录页 + 注册页共 3 个密码框）
- 登录页表单卡片容器分组（+MaterialCardView）
- 注册页密码实时强度指示器
- 注册页邮箱键盘输入类型
- 注册页 Toolbar 返回键绑定
- 欢迎页退出按钮红色警告色
- 空待办列表插画 + 引导文字
- 列表项 CardView 圆角提升（8dp → 12dp）
- 待办编辑页裸露 EditText → TextInputLayout（移除冗余标签）
- 待办编辑页内容框 minLines=5
- 待办编辑页添加内联错误校验
- 待办详情页状态标签语义颜色（完成=绿，未完成=灰/橙）
- 待办详情页内容区域 MaterialCardView 包裹

## 影响

- **影响页面**：全部 6 个 Activity 布局 XML + 2 个 Java 文件
- **当前分支**：`feature/ui-polish`（已有 UI 审查文档）
- **风险**：布局重构（编辑页 TextInputLayout）可能影响现有功能，需编译验证

## 新增需求

### 需求: 密码可见性切换

系统**应当**在登录页密码框和注册页的两个密码框上提供密码可见性切换按钮。

#### 场景: 用户点击切换图标
- **WHEN** 用户在密码框中输入密码
- **THEN** 输入框右侧显示眼睛图标
- **WHEN** 用户点击眼睛图标
- **THEN** 密码明文/密文切换显示

### 需求: 空状态插画

系统**应当**在待办列表为空时显示插画 + 引导文字，而非仅有文字。

#### 场景: 用户首次打开待办列表
- **WHEN** 待办列表为空
- **THEN** 显示一个插画（在 `tv_empty_hint` 上方）
- **AND** 引导文字改为"还没有待办事项\n点击下方 + 按钮创建一个"

### 需求: 表单内联错误校验

待办编辑页**应当**在标题为空时显示内联错误，而非仅弹 Toast。

#### 场景: 用户提交空标题
- **WHEN** 用户点击保存且标题为空
- **THEN** 标题输入框边框变红
- **AND** 下方显示错误文字"标题不能为空"

### 需求: 状态标签语义颜色

待办详情页**应当**根据完成状态显示不同颜色的状态标签。

#### 场景: 查看不同状态待办
- **WHEN** 待办已完成
- **THEN** 状态标签显示绿色
- **WHEN** 待办未完成
- **THEN** 状态标签显示灰色或橙色

## 修改需求

### 需求: 表单卡片分组

登录页的输入框和 CheckBox **应当**被包裹在 MaterialCardView 中，与操作按钮（登录/注册）形成两个视觉区块。

### 需求: 编辑页输入框升级

待办编辑页的标题/内容输入框**应当**从裸露 EditText 升级为 M3 TextInputLayout（OutlinedBox 风格），移除冗余的 `label_title` / `label_content` 标签。

### 需求: 编辑页内容框高度

`et_content` **应当**设置 `minLines="5"`，内容从顶部开始排列（`gravity="top"`）。

### 需求: 注册页邮箱键盘类型

`et_email` **应当**使用 `inputType="textEmailAddress"` 以显示带 `@` 和 `.com` 快捷键的键盘。

### 需求: 注册页 Toolbar 返回键绑定

`RegisterActivity.java` **应当**为 Toolbar 注册 `setNavigationOnClickListener`。

### 需求: 欢迎页退出按钮警告色

`btn_logout` **应当**使用红色描边/文字以表示危险操作。

### 需求: 列表项圆角提升

`item_todo.xml` 的 CardView **应当**使用 `app:cardCornerRadius="12dp"`。
