# 实训 4　注册与登录功能的实现

> 课程：移动应用开发（Android）实训
> 主题：在实训 3 的数据库连接之上，接入真正的注册与登录业务
> 前置：[实训 3 Android 与 MySQL 的连接](./实训3-Android与MySQL的连接.md)
> 后续：[实训 5 记住密码](./实训5-记住密码.md)

## 项目描述

项目要求 Android Studio 通过 MySQL 数据库实现**登录、注册**功能。

## 项目目标

- 熟练 MySQL 与 Android 应用的开发的相关步骤与设置
- 熟练 Android 在 MySQL 中对数据库的增、删、查、改的相关操作

## 实践步骤

### 一、连接 MySQL 数据库服务器

1. 打开 **Navicat**。
2. 单击"连接"菜单，然后选择"MySQL"。
3. 在"连接"对话框中输入连接信息。

> ⚠️ 本节与 [实训 3](./实训3-Android与MySQL的连接.md) 完全重复；如果已经在实训 3 中完成连接，可直接跳过。

### 二、创建数据库与数据表

1. 创建新的数据库 `androidPT`。
2. 创建 `Userinfo` 表，`id` 为主键并自动递增。

> ⚠️ 同上，与 [实训 3](./实训3-Android与MySQL的连接.md) 重复。

### 三、Android Studio 配置

#### 1. 添加 MySQL 依赖

在 `build.gradle` 中添加：

```gradle
implementation("mysql:mysql-connector-java:5.1.49")
```

#### 2. 添加网络权限

在 `AndroidManifest.xml` 中：

```xml
<uses-permission android:name="android.permission.INTERNET" />
```

#### 3. 代码包结构

> ⚠️ 本实训在原文包名规划上**与实训 3 略有不同**：原文只写了 `dbunit` 一个包，没有把 `bean` / `dao` 与其一并列出，但后文又提到了 `User.java` 和 `UserDao.java`。建议沿用实训 3 的三包结构：
>
> ```
> java/<your.package>/
> ├── dbunit/JdbcHelper.java
> ├── bean/User.java
> └── dao/UserDao.java
> ```

| 类 | 职责 |
|----|------|
| `JdbcHelper.java` | 取得 `Connection` |
| `User.java` | 用户实体（id、username、password 等） |
| `UserDao.java` | 封装注册、登录对应的 SQL（INSERT / SELECT） |

> 📌 `UserDao` 中的连接代码用于测试数据库的连接是否通畅，应放在子线程中。

### 四、实现注册界面功能（自主实现）

要点：

- 收集用户输入的账号、密码。
- 在 `UserDao` 中提供 `register(User user)` 方法，**先 SELECT 判重再 INSERT**。
- 成功 / 失败都要通过 `Toast` 或 `Handler` 反馈给用户。

### 五、实现登录界面功能（自主实现）

要点：

- 在 `UserDao` 中提供 `login(String username, String password)` 方法，返回 `User` 或 `null`。
- 登录成功 → 跳到欢迎页（见下一步）。
- 登录失败 → `Toast` 提示。

### 六、登录成功后跳到欢迎页面（自主实现）

- 新建 `WelcomeActivity` 作为登录成功后的目标页。
- 通过 `Intent` 携带用户名跳过去并在欢迎页上展示。

> 📌 这里的跳转在 [实训 6 自动登录](./实训6-自动登录.md) 中会被改造成带结果回传（`startActivityForResult`）的形式。

## 作业要求

| 步骤 | 状态 |
|------|------|
| 连接到 MySQL | 自查 |
| 实现 `UserDao.register` | 自主实现 |
| 实现 `UserDao.login` | 自主实现 |
| 注册页 UI / 逻辑打通 | 自主实现 |
| 登录页 UI / 逻辑打通 | 自主实现 |
| 跳转到欢迎页 | 自主实现 |
