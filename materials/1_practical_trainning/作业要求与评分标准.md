## 一、功能完整性

### 1.1 注册功能（必做）

- [ ] 用户可输入**用户名、密码、邮箱**等基本信息
- [ ] 校验输入信息的合法性
  - [ ] 用户名是否已存在
  - [ ] 密码强度（长度、复杂度等）
  - [ ] 邮箱格式
- [ ] 注册成功 / 失败有明确反馈（Toast / 页面跳转 / 提示信息）

**对应实训**：[实训 1](./实训1-登录与注册页面.md) UI → [实训 3](./实训3-Android与MySQL的连接.md) 数据库 → [实训 4](./实训4-注册与登录功能实现.md) 业务

**实现要点**：

```sql
-- 注册时先 SELECT 判重再 INSERT
SELECT * FROM Userinfo WHERE username = ?;
INSERT INTO Userinfo (username, password, email) VALUES (?, ?, ?);
```

### 1.2 登录功能（必做）

- [ ] 校验**用户名 + 密码**的正确性
- [ ] 登录成功 → 跳转到欢迎页
- [ ] 登录失败 → 明确错误提示（账号不存在 / 密码错误 / 账号为空 等）
- [ ] 登录失败次数过多时的处理（**可选扩展**，不强制）

**对应实训**：[实训 2](./实训2-Activity跳转与Toast.md) 占位逻辑 → [实训 4](./实训4-注册与登录功能实现.md) 真实业务

**实现要点**：

```sql
SELECT * FROM Userinfo WHERE username = ? AND password = ?;
```

### 1.3 记住密码 / 自动登录（可选加分项）

- [ ] "记住密码"勾选后，下次启动自动回填账号密码
- [ ] "自动登录"勾选后，下次启动直接进入欢迎页
- [ ] 勾选状态联动合理（见 [实训 6](./实训6-自动登录.md)）

**对应实训**：[实训 5](./实训5-记住密码.md) + [实训 6](./实训6-自动登录.md)

> 📌 这两项是**可选实现**的加分项。功能没做不影响"必做"部分的得分，但做了一定要正确、不能出 bug，否则会扣分。

## 二、界面设计

- [ ] 布局合理、美观易用，符合 Android Material Design 规范
- [ ] 注册 / 登录界面有**清晰、友好的提示信息**
  - [ ] 占位提示（`hint`）
  - [ ] 输入不合法时的红色 / Toast 错误提示
  - [ ] 成功 / 失败的明确反馈
- [ ] 不同屏幕尺寸下能正常显示（避免硬编码像素）
- [ ] 适当的图标、间距、配色

**对应实训**：[实训 1 登录与注册页面](./实训1-登录与注册页面.md)（colors / style / themes / drawable 资源）

**实现建议**：

- 使用 `dp` / `sp` 而非 `px`
- 复用 `MyBtnStyle` / `MyEditStyle` 这类自定义样式
- 颜色集中在 `colors.xml`，便于统一调整
- 按钮 / 输入框使用 `selector` / `shape` drawable 处理状态

## 三、代码规范

- [ ] 代码结构清晰：包结构合理（参考 `dbunit` / `bean` / `dao` 三包拆分）
- [ ] 逻辑明确：方法职责单一，不过长（> 50 行的方法建议拆）
- [ ] 注释详细：关键方法、关键 SQL、关键算法都要有
- [ ] 命名规范：
  - 类名：`PascalCase`（如 `UserDao`）
  - 方法 / 变量名：`camelCase`（如 `login` / `userName`）
  - 常量：`UPPER_SNAKE_CASE`
  - 资源文件：`snake_case`（如 `activity_main.xml`）
- [ ] 遵循 Android 开发最佳实践：
  - [ ] 数据库操作在**子线程**中执行（见 [实训 3](./实训3-Android与MySQL的连接.md)）
  - [ ] UI 更新切回主线程（`runOnUiThread` / `Handler`）
  - [ ] 不在主线程做 IO / 网络
  - [ ] `Activity` 跳转用 `Intent`，必要时用 `startActivityForResult`（见 [实训 6](./实训6-自动登录.md)）
