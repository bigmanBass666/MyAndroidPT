## 三、遇到的问题及解决办法

> 填写建议：把实训 3 ~ 6 中实际踩到的坑写成条目，并给出**解决办法**。下面是一些本套实训中**已知的坑**，可以直接对照自己是否遇到过：

| 问题现象 | 原因 | 解决办法 |
|---------|------|---------|
| 报 `Communications link failure` | Android 端用了 `localhost` 连 MySQL | 必须用本机或服务器的 **IP** 形式连接 |
| Navicat 用 IP 也连不上 MySQL | MySQL 没有给 `root@%` 授权 | 执行授权 SQL（MySQL 5.x / 8.0 写法不同，详见 [实训 3](./实训3-Android与MySQL的连接.md)） |
| 一访问数据库就 crash | 数据库连接跑在主线程 | 用 `new Thread(...).start()` 包起来；UI 更新用 `runOnUiThread` |
| 注册时插入中文乱码 | JDBC URL 没指定字符集 | URL 加上 `?useUnicode=true&characterEncoding=UTF-8` |
| 勾选"自动登录"但下次打开还要手动登录 | SharedPreferences 没在登录成功时写入 `isAutoLogin` | 检查 `editor.putBoolean("isAutoLogin", true).apply()` 是否在登录成功分支里 |
| 退出登录后"自动登录"还是勾着 | 退出登录时只 finish 没清标志 | 在退出按钮回调里 `putBoolean("isAutoLogin", false).apply()` |
| 注册后跳回登录页，账号没回填 | 用了 `startActivity` 没用 `startActivityForResult`，且 `setResult` 没调用 | 见 [实训 6 步骤 4](./实训6-自动登录.md) |
