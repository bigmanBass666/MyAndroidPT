## 1.2 题目

> **基于安卓的简易待办应用开发**

## 1.3 功能要求（必须全部实现）

| # | 功能 | 行为说明 |
| --- | --- | --- |
| F1 | 待办创建 | 用户能方便地创建新待办，支持输入文本 |
| F2 | 待办查看 | 列出所有已保存的待办；点击列表项查看详情 |
| F3 | 待办编辑 | 对已有待办进行修改编辑 |
| F4 | 待办删除 | 删除不再需要的待办 |
| F5 | 待办状态切换 | 切换"已完成 / 未完成"两种状态 |

> 以上 F1–F5 在 `02-grading-rubric.md` 中对应"功能实现"维度的全部得分区间；任一功能缺失会拉低档位。

## 1.4 界面要求

- 界面**简洁直观**，按钮布局合理
- 操作路径短：创建 / 查看 / 编辑 / 删除的入口清晰
- 遵循**安卓基本设计风格与交互规范**（Material 习惯、触控反馈、返回栈合理）

## 1.5 技术要求（硬性红线）

- **语言**：Java（安卓原生）
- **组件**：使用 Activity 组件实现各功能界面
- **存储**：使用 SQLite 存储待办数据
- **兼容性**：在不同分辨率、不同安卓版本的设备上能正常运行，无明显兼容性问题

> 提示：如果用 Kotlin 或 Room/其它 ORM 替代，会偏离"技术要求"原文；除非有充分理由并写入报告说明，否则建议严格按 Java + Activity + SQLite 实现。

## 1.6 提交方式

需提交 **2 个文件**，均通过邮件提交至老师指定邮箱：

| 类别 | 文件 | 命名规范 | 格式 |
| --- | --- | --- | --- |
| 代码 | 整个安卓项目源码 | `学号_姓名_简易待办代码.zip` | zip 压缩包 |

> ⚠️ 命名必须严格遵循"学号_姓名_..."模式，错误命名可能影响成绩登记。

### 1.6.1 报告应包含的章节（详见 `03-report-template.md`）
