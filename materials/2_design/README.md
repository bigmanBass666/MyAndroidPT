# 课程设计材料总览

本目录由**三份原始文档**（一份 .doc 课程试卷 + 一份 .docx 报告模板 + 一份 .md 课堂派公告）经内容合并与有机重构而来，面向 AI 阅读与理解场景而编排。

## 一、文件清单

| 文件 | 角色 | 何时读它 |
| --- | --- | --- |
| `01-assignment.md` | 课程考核任务书：考务信息 + 题目 + 功能/界面/技术要求 + 提交方式 | 需要了解"做什么、怎么交"时 |
| `02-grading-rubric.md` | 评分细则：功能 50 / 界面 20 / 代码 20 / 文档 10 四档打分区间 | 需要知道"怎么被评分、哪些点拿分"时 |
| `03-report-template.md` | 课程设计报告模板：7 个一级章节的写作结构与填空指引 | 需要撰写报告、组织章节时 |
| `README.md` | 本文件，索引与关键元数据 | 入口 |

## 二、关键事实速查

- **学校 / 学院**：华南农业大学珠江学院 · 人工智能学院
- **课程**：移动应用开发（Android）
- **考试类型**：开卷 · 个人作业
- **提交平台**：**课堂派 · 课程设计考核**（不是邮箱）
- **提交窗口**：**2026-06-08 09:15 ~ 2026-06-26 00:00**（允许超时，但计 100 分查重）
- **总分构成**：平时成绩 40% + 期末课程设计 60%
- **考核题目**：基于安卓的简易待办应用开发
- **技术栈硬性要求**：安卓原生 + **Java** + Activity 组件 + SQLite 存储
- **提交物**（3 份）：
  - 代码：`学号_姓名_简易待办代码.zip`
  - 报告：`学号_姓名_简易待办课程设计报告.docx`
  - 视频：`学号_姓名_展示视频`（**3 分钟以内、本人声音**）

## 四、给 AI 阅读的建议路径

1. **先读本 README** 把握全貌
2. **读 `01-assignment.md`** 锁定题目硬约束 + 当前最新的提交规则（Java / Activity / SQLite 三条技术红线 **+** 课堂派 3 份提交物 **+** 3 分钟带声音的视频）
3. **读 `02-grading-rubric.md`** 把"分值 → 行为"对齐，知道每分花在哪里最划算
4. **读 `03-report-template.md`** 按章节骨架填报告，注意"问题及解决办法"是双问题模板

两份原始任务描述（试卷 A 卷 + 课堂派公告）存在以下差异，已在合并时按"**以课堂派公告版为准**"处理：
