**建议覆盖**：

## 三、设计思路

本节包含两个子模块，是原模板保留的二级标题。

### 3.1 模块划分

**写什么**：把整个 App 拆成几个模块 / 包 / 界面，并说明各自职责。

**建议模板**（按需调整）：

| 模块 / 类 | 职责 |
| --- | --- |
| `MainActivity` | 待办列表展示 + 入口 |
| `AddTodoActivity` | 创建待办 |
| `EditTodoActivity` 或同界面 | 编辑待办 |
| `TodoDetailActivity` | 查看详情 |
| `TodoDbHelper`（SQLiteOpenHelper） | 数据库建表、CRUD |
| `Todo` 实体类 | 字段：id、title、content、status、createdAt 等 |
| 布局 XML | 列表 item、详情、表单 |

### 3.2 涉及知识点

**写什么**：把代码里**实际用到**的安卓知识点列出来，标明"用在哪"。

**建议覆盖**（按本课程常用知识圈）：

- Activity 生命周期、Intent 跳转与回传
- ListView / RecyclerView 列表展示与点击事件
- SQLite（SQLiteOpenHelper + SQLiteDatabase + Cursor）
- 布局（XML、ConstraintLayout/LinearLayout）
- 资源（strings.xml、colors.xml、drawable）
- 简单的事件处理（Button 点击、长按删除等）

## 四、功能实现步骤

**写什么**：按时间顺序写"我是怎么一步步把 App 做出来的"。建议按模块或按功能来分段。

**建议结构**：

1. 项目创建与目录结构
2. 数据库设计（表结构 + 建表 SQL 片段）
3. 实体类与基础 CRUD 方法
4. 列表页（MainActivity）实现 → 截图占位
5. 创建功能（AddTodoActivity）实现 → 截图占位
6. 查看 / 编辑 / 删除功能实现 → 截图占位
7. 状态切换功能实现 → 截图占位
8. 调试与兼容性自查

> 关键代码片段（建表 SQL、Adapter 关键逻辑、Activity 间传值等）建议贴 5–15 行的小片段并加注释，避免大段贴代码。

## 五、遇到的问题及解决办法

> 原模板给了 **2 个问题** 的写作槽位，下面是双问题模板。

### 问题 1：（自拟标题，如"列表点击无响应"）

- **原因分析**：描述问题具体现象及原因。
  - 现象：（报错信息 / 用户反馈 / 操作结果）
  - 原因：（定位到的代码或环境问题）
- **解决方案**：说明解决过程和最终方法。
  - 排查过程：（一步步怎么定位）
  - 最终方法：（具体改了什么 / 加了什么）

### 问题 2：（自拟标题，如"应用在低版本安卓上崩溃"）

- **原因分析**：描述问题具体现象及原因。
- **解决方案**：说明解决过程和最终方法。

> 建议选**真实遇到过**的问题，不要编造。可考虑：ListView 点击无响应 / SQLite 关闭异常 / Activity 跳转传值丢失 / 屏幕旋转后数据丢失 / 最低 SDK 兼容 等。
