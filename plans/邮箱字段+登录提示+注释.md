# 邮箱字段 + 登录失败提示细化 + 关键注释计划

> 背景：对照评分标准（作业要求与评分标准.md），当前代码有两处明确缺口：邮箱字段缺失（1.1）、登录失败提示未细分（1.2），以及关键 SQL/方法缺少注释（3.x）。

---

## 改动 1：注册流程 — 增加邮箱字段

### 为什么做
评分标准 1.1 要求"邮箱格式校验"。当前 RegisterActivity 和 User 实体均无 email 字段。

### 涉及文件

| 文件 | 改动点 |
|------|--------|
| `bean/User.java` | 新增 `email` 字段（String），更新构造方法、toString |
| `activity_register.xml` | 新增邮箱 EditText (`et_email`)，包含于响应式布局 |
| `RegisterActivity.java` | 加邮箱输入获取 + 格式校验（正则）→ 校验不通过 Toast 提示；注册时把 email 写入 User → UserDao 存储 email |
| `dbunit/UserDBHelper.java` | CREATE TABLE 语句加 email 列；onCreate/onUpgrade 加 email 字段 |
| `dao/UserDao.java` | register() 的 INSERT SQL 加 email 列；register() 参数不变（User 已含 email） |

### 关键决策
- **邮箱校验正则**：使用 `^[A-Za-z0-9+_.-]+@[A-Za-z0-9.-]+$`，这是 Android 标准 `Patterns.EMAIL_ADDRESS` 的等价简化版，避免引入新 API 兼容问题。
- **onUpgrade 处理**：直接 DROP 重建（项目全局约定如此，TodoDBHelper 同理），已有用户数据邮箱留空。
- **字段非空约束**：参考 User 表的 `userName` 和 `password` 均为 NOT NULL，email 也设为 NOT NULL，注册时必须填入合法邮箱。
- **注册结果回传**：`setResult` 时附带 email 字段，MainActivity `onActivityResult` 中回填到三个输入框（账号/密码/邮箱）。

## 改动 2：登录失败提示细化

### 为什么做
评分标准 1.2 要求"登录失败 → 明确错误提示（账号不存在 / 密码错误 / 账号为空 等）"。当前只有一句 Toast "登录失败"。

### 涉及文件

| 文件 | 改动点 |
|------|--------|
| `dao/UserDao.java` | `login()` 方法改为返回区分结果枚举，新增一个内部类 `LoginResult`（含状态码 + message），状态码区分：`OK`、`USER_NOT_FOUND`、`WRONG_PASSWORD`、`EMPTY_INPUT` |
| `MainActivity.java` | 登录按钮点击逻辑中，根据 `LoginResult` 显示不同 Toast 文案；空输入校验仍保留在 Activity 层（快速失败） |

### 关键决策
- **异常改为业务回传**：不抛异常，用枚举区分场景。评分维度看重"明确错误提示"，枚举方式对评分员更直观。
- **空输入校验位置**：保留在 Activity 层（点击登录按钮时先 check isEmpty），不走到 DAO 层。

## 改动 3：关键方法/SQL 注释补充

### 为什么做
评分标准 3.x 注释要求："关键方法、关键 SQL、关键算法都要有注释"。

### 涉及文件

| 文件 | 注释位置 |
|------|---------|
| `dao/UserDao.java` | 每个方法头部（功能描述 + 参数 + 返回）；每段 SQL 上方加行注释说明用途 |
| `dbunit/UserDBHelper.java` | onCreate/onUpgrade 中的每段 DDL 加注释；email 字段新增同步注释 |
| `RegisterActivity.java` | 邮箱校验正则加注释；registerForResult 回传逻辑加注释 |

### 关键决策
- **只加关键注释，不全文遍历**：扫描后确认 UserDao 约 170 行，UserDBHelper 约 120 行，重点覆盖 DAO 方法和 DDL 语句。
- **注释风格**：现有代码部分区域已有中文注释，统一沿用中文行注释（`//`），不加 Javadoc 风格的块注释（`/** */`），保持与项目整体风格一致。
